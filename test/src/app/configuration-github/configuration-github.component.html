Configuration of Github

As a user, I want to be able to view my fetched user stories from Jira and select the user stories so that I can access and review the details of the stories related to my project.

Acceptance Criteria

  • User stories should be successfully fetched from Jira API
  • User stories should be displayed in a readable and organized manner in the front-end application.
  • User stories should be displayed in real-time without any delay or lag.

Acceptance Testing

  • Verify that user stories are successfully fetched from Jira API
  • Verify that the fetched user stories are displayed correctly in the front-end application with the necessary details such as the story title, description, and assigned team member.
  • Verify that the user stories are displayed in real-time without any delay or lag.

Definition of Done

  • The user stories have been successfully fetched from Jira API and displayed in the front-end application.
  • The user stories are displayed in a readable and organized manner with the necessary details.
  • The user stories are displayed in real-time without any delay or lag.
  • All acceptance criteria and acceptance testing have been successfully met and verified.
